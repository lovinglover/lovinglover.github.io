引用

引用与指针的区别

代码中int &b = a语句就是声明并初始化一个引用，从打印结果中可以看到：

  1. 引用复制了变量a的地址，它没有自己的地址，a和b共用一块内存，引用只是变量的一个别名，此时对b做修改，a也会跟着一起改变

    类似于一个房间有两个门，你对房间内部做了重新装修，但房间还是那个房间，你从哪个门进去看到的都是一样的

关于引用需要注意的

  1. 必须先有一个变量，然后针对这个变量建立引用，在声明引用的同使必须进行初始化，表明是对哪个变量额引用，除了在函数中作为形参出现

  2. 引用一旦声明并初始化，便不可更改，即引用相当于一个常量

    注意：这不是将b作为c的引用的意思

  3. 不能建立引用数组

new的生命周期

new出来的变量，如果你不主动 delete 掉这段申请的内存的话，它会一直存在，直到进程结束后系统会回收掉这段资源；而如果你delete掉这段申请的内存，则这段申请到的内存的生命周期为从你new（申请一段内存）到你delete（释放掉这段内存）这段时间。

使用这些内置的字符串处理函数需要包括头文件,使用语句#include <string>

连接函数strcat

该函数的定义原型为：

char * strcat(char * destination, const char * source);

函数的作用是将第二个字符串追加到第一个字符串后面，不会改变第二个字符串，得到的结果是destination = destination + source，函数返回第一个字符串的首地址。

注意：这里必须保证destination的总长度足够

需要满足总长度要大于等于二者字符串长度之和再加1，因为还有一个'\0'结束符，即这里的最小长度是11，而不是10

拷贝函数strcpy

定义原型：

char * strcpy ( char * destination, const char * source );

函数作用是source的内容拷贝到destination，包括'\0'结束符，所以会”覆盖“掉destination的内容，但不是真的覆盖，只是因为字符串认为遇到'\0'就是结束，而函数strcpy也会拷贝'\0'

看截图可知，des并没有被全部覆盖，只是strcpy函数将'\0'一起拷贝过来，导致des提前结束，从而只输出了test。

比较函数strcmp

定义原型为：

int strcmp ( const char * str1, const char * str2 );

函数作用是逐个字符对比两个字符串，直到比出大小或者到结束，比较依据是每个字符的ASCII码。若str1>str2，返回1；若str1<str2，返回-1；若str1=str2，返回0。

可以看到前面都是一样的字符串，长的大于短的

求长度函数strlen和占字节函数sizeof

strlen的定义原型：

size_t strlen ( const char * str );

求得字符串的长度，到'\0'截止，但是不包括'\0'

可以看到通过strlen得到的是字符串的实际大小，而sizeof得到的这个数组的大小，每个char占一个字节。
