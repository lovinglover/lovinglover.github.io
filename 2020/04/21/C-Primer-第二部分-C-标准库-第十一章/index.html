第十一章 关联容器

关联容器支持高效的关键字查找和访问。两个主要的关联容器(associative-container)类型是map和set。map中元素是关键字-值对(键值对key-value)：关键字起到索引作用， 值表示与所以关联的数据。set中每个元素只包含一个关键字，set支持高效的关键字查询-给定关键字是否在set中。

标准库提供8个关联容器，map和multimap定义在头文件map中；set和multiset定义在头文件set中；无序容器定义在头文件unordered_map和unordered_set中。

关联容器 说明
map 保存键值对
set 关键字即值，只保存关键字
multimap 关键字可以重复出现的map
multiset 关键字可以重复出现的set
unordered_map 用哈希函数组织的map
unordered_set 用哈希函数组织的set
unordered_multimap 关键字可以重复出现的用哈希组织的map
unordered_multiset 关键字可以重复出现的用哈希组织的set

11.2 关联容器概述

关联容器(有序和无序)都支持普通容器的操作，但不支持顺序容器的位置相关操作，如push_front这种。也不接受构造函数或插入函数这样接受一个元素值和一个数量值的操作。

关联容器的迭代器都是双向的。

11.2.2 关键字类型的要求

有序的关联容器中，关键字必须能够比较，默认使用其类型的<运算符，若自定义，必须是严格弱序(strict weak ordering)，可以看作”小于等于“

  • 两个关键字不能同时小于等于对方
  • 若k1”小于等于“k2，k2”小于等于“k3，k1必须”小于等于“k3
  • 若存在两个关键字都不”小于等于“对方，称为”等价“

若两个关键字”等价”，视为相等，只能有一个元素与这2个关键字关联，可以用二者之一来访问这个元素。

11.2.3 pair类型

一个pair保存2个数据成员。类似容器，pair是一个用来生成特定类型的模板。

pair与map的关系：map中的元素是pair，一个pair保存2个数据

标准库中pair的操作

操作 说明
pair<T1, T2> p; p是个pair，执行默认初始化
pair<T1, T2> p(v1, v2);
pair<T1, T2> p{v1, v2};		 用v1和v2初始化
make_pair(v1, v2) 用v1和v2初始化，类型从v1和v2推断
p.first 返回名为first公有数据成员(对应map中的key)
p.second 返回名为second公有数据成员(对应map中的value)
关系运算符 按字典序。p1.first<p2.first或!(p2.first<p1.first)&&p1.second<p1.second;p1<p2
p1==p2;p1!=p2 当且仅当 p1.first==p2.first&&p1.second==p2.second，相等

11.3 关联容器

额外类型别名 说明
key_type 此容器类型的关键字类型
mapped_type 每个关键字关联的类型，只适用于map
value_type 对于set，与key_type相同
对于map，为pair<const key_type, mapped_type>

map中，每个元素是一个pair对象，无法修改关键字

11.3.1 关联容器迭代器

当解引用关联容器的迭代器时，得到类型为容器的value_type的值的引用。对于map，value_type是pair类型，其first成员是const的，不能修改。对于set，只能读取，不能修改。

map和set都支持begin和end操作。

一般不对关联容器使用泛型算法，因为关键词是const的。

11.3.2 添加元素

insert成员向容器添加一个元素或一个元素范围

操作 说明
c.insert(v)
c.emplace(args)		 v是value_type类型的对象，args用来构造元素
对于map和set，只有当元素关键字不在c中才插入元素，返回一个pair包含一个迭代器，指向具体指定关键字的元素，以及一个是否成功的bool
对于multimap和multiset，总会插入或构造指定元素，返回指向新元素的迭代器
c.insert(b, e)
c.insert(il)		 b和e是迭代器，表示一个c::value_type类型的范围，il是这种值的花括号列表，返回void
对于map和set，直插入关键字不在c中的元素
对于multimap和multiset，会插入范围中的每个元素
c.insert(p, v)
c.emplace(p, args)		 类似insert(v)，将迭代器p作为提示，指出从哪里开始搜索新元素该存储的位置
返回迭代器，指向具有给定关键字的元素

使用insert统计单词在输入中出现的次数

11.3.3 删除元素

操作 说明
c.erase(k) 从c中删除关键字为k的元素。
返回一个size_type，说明被删除元素的数量
c.erase(p) 从c中删除迭代器p指定的元素
p不能是c.end()
返回指向p之后元素的迭代器，若p是尾元素，返回c.end()
c.erase(b, e) 删除b和e表示的范围内的元素，返回e

11.3.4 map的下标操作

map和unordered_map提供了下标运算符和对应的at函数。

操作 说明
c[k] 返回关键字为k的元素
若k不在c中，添加关键字为k的元素，对其进行初始化
c.at(k) 访问关键字为k的元素，带参数检查
若k不在c中，抛出out_of_range异常

11.3.5 访问元素

操作 说明
c.find(k) 返回一个迭代器，指向第一个关键字为k的元素
若k不在容器，返回尾后迭代器
c.count(k) 返回关键字为k的元素数量
对不允许重复的，永远是0或1
c.lower_bound(k) 返回迭代器，指向第一个关键字不小于k的元素
c.upper_bound(k) 返回迭代器，指向第一个关键字大于k的元素
c.equal_range(k) 返回迭代器pair，表示关键字等于k的元素范围
若k不存在，pair两个成员都是c.end()

其中lower_bound和upper_bound不适用于无序容器

11.4 无序容器

新标准定义了4个无序关联容器(unordered associative container)，使用哈希函数和关键字类型的==运算符。

无序容器在存储上组织为一组桶，每个桶保存0个或多个元素，容器使用哈希函数将元素映射到桶。

桶管理操作

桶接口
c.bucket_count() 正在使用的桶的数目
c.max_bucket_count() 容器最多能容纳多少桶
c,bucket_size(n) 第n个桶有多少元素
c.bucket(k) 关键字为k的元素在哪个桶
桶迭代
local_iterator 用来访问桶中元素的迭代器类型
const_local_iterator 桶迭代器的const版本
c.begin()、c.end() 首元素迭代器和尾后迭代器
c.cbegin()、c.cend() const版本的
哈希策略
c.load_factor() 每个桶的平均元素量，返回float
c.max_load_factor() c试图维护桶的平均大小，返回float。c会在需要时添加新的桶，使
load_factor<=max_load_factor
c.rehash(n) 重组存储，是的bucket_count>=n
且bucket_count>size/max_load_factor
c.reserve(n) 重组存储，使得c可以保存n个元素且不用rehash

无序容器还提供了与有序容器相同的操作(find、insert等等)，也存在允许重复关键字版本。

无序容器的性能依赖于哈希函数的质量和桶的数量与大小。
