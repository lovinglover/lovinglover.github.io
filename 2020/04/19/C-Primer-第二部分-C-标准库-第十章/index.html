C++ Primer 第二部分 C++标准库

第十章 泛型算法

标准库定义了一组泛型算法(generic algorithm)，称为“算法”因为他们实现了一些经典算法的公共接口，如搜索和排序；称为“泛型”因为可以用于不同类型的元素和多种容器类型。

10.1 概述

大多数算法定义在头文件algorithm中，还在numeric头文件中定义了一组数值泛型算法

一般情况下，算法不直接操作容器，而是遍历由两个迭代器指定的一个范围。

如算法find

find前两个参数指定范围，第三个参数表示要寻找的元素。如存在，返回指向第一个等于给定值的元素的迭代器；不存在返回第二个参数，即范围结束的迭代器。

算法依赖于迭代器，永远不会执行容器的操作。

10.2 初识泛型算法

除少数外，标准库算法都对一个范围内的元素进行操作，这个范围称为“输入范围”。这个范围由两个参数表示，分别指向要处理的第一个元素，和要处理的最后一个元素之后的位置。

10.2.1 只读算法

对于只读算法，通常最好用cbegin()和cend()，除非有特殊需求。

一些算法只读取输入范围内的值，不改变元素。如：find、count、accumulate(定义在numeric头文件)

equal用来确定两个序列的值是否相同(依次比较对应位置上的值)。接受三个参：前两个表示范围，第三个表示第二个序列的首元素(第二个序列至少和第一个一样长)

10.2.2 写容器元素的算法

序列原大小不能小于写入的数目，因为算法不执行容器操作，无法改变容器大小。

fill将第三个参数的值赋予输入范围的每个元素。

fill_n接受三个参数：一个迭代器，一个计数值、一个值。将给定值赋予指定迭代器指定的元素开始的指定个元素。

back_inserter接受一个指向容器的引用，返回一个与改容器绑定的插入迭代器(见后面)，定义在头文件iterator中。通过这个迭代器赋值，赋值运算符会调用push_back将元素添加到容器中

也可以使用back_inserter创建一个迭代器，作为算法的目的位置

copy将输入范围的值写入目标序列，返回目的迭代器递增后的值。

replace替换给定值

replace_copy接受额外第三个迭代器，指出调整后序列的保存位置

list并未发生改变，改变后的序列放在ivec中

10.2.3 重排容器元素的算法

sort依据元素类型的<运算符实现排序

unique将相邻的重复项消除，返回一个指向不重复范围末尾元素下一个的迭代器

10.3 定制操作

这些算法允许提供自己定义的操作代替默认运算符

10.3.1 向算法传递函数

使用sort重载版本，接受第三个参数，一个谓词(predicate)

谓词是一个可调用的表达式，返回结果是一个能用作条件的值。标准库算法使用的谓词分为两类：一元谓词(unary predicate，只接受单一参数)和二元谓词(binary predicate，接受2个参数)。算法调用的元素类型必须可以转换为谓词的参数类型。

这时sort会按照长短来排序，但是等长字符是无序的

使用stable_sort使等长字符间按字典序排列

10.3.2 lambda表达式

一个lambda表达式表示一个可以调用的代码单元，理解为一个未命名的内联函数。一个lambda具有返回类型、参数列表和函数体，但可以定义在函数内部。

[capture lsit](parameter lsit) -> return type {function body}

其中，capture list(捕获列表)是一个lambda所在函数中定义的局部变量的列表(通常为空)；return type、parameter list和function body与普通函数一样。lambda必须有捕获列表和函数体。

[] { return 42; }

实现biggies函数，打印大于给定长度的单词

10.3.3 lambda捕获和返回

捕获方式有三种：值捕获、引用捕获、隐式捕获

可变lambda

默认情况下，对于一个值被拷贝的变量，lambda不会改变其值。若希望改变，需要关键字mutable

指定lambda返回类型

默认情况下，如果一个lambda包含return之外的任何语句，返回类型假定为void，被推断返回void的lambda不返回值

transform接受三个迭代器和一个可调用对象。前两个表示输入序列，第三个迭代器表示目的位置，函数对输入序列中每个元素调用可调用对象，将结果写到目的位置。

10.3.4 参数绑定

如find_if的可调用参数必须接受单一对象，要想使用函数来替代lambda，必须解决传递形参的问题。

使用bind绑定check_size()的参数

bind定义在头文件functional中

auto newCallable = bind(callable, arg_list);

newCallable是一个可调用对象，arg_list是逗号分隔的参数列表，调用newCallable时，newCallable会调用callable，并传递arg_list中的参数。

arg_list中有这样的参数：_n。表示占位，说明newCallable接受的参数是newCallable的第几个参数。

使用check_size的find_if

bind将check_size的第二个参数绑定到sz上。

使用_1需要使用命名空间

可以使用bind实现重排参数顺序

若要绑定引用，使用ref，ref返回一个对象包含给定的引用

10.4 再探迭代器

除了为容器定义的迭代器，还定义了其他迭代器

  • 插入迭代器(insert iterator)：被绑定到一个容器上，用来向容器插入元素
  • 流迭代器(stream iterator)：被绑定到输入或输出流，可用来遍历相关的IO流
  • 反向迭代器(reverse iterator)：除了forward_list都有反向迭代器，自增自减含义相反
  • 移动迭代器(move iterator)：不拷贝其中元素，移动

10.4.1 插入迭代器

接受一个容器，调用容器操作，向容器中插入元素

三种类型，插入位置不同：

  • back_insert：使用push_back
  • front_insert：使用push_front
  • inserter(c, iter)：使用insert，接受第二个参数，指向给定容器的迭代器。元素被插入到给定迭代器指向的元素之前位置

10.4.2 iostream迭代器

istream_iterator读取输入流

ostream_iterator向输出流写数据

创建istream_iterator时可以绑定一个流，也可以使用默认初始化，这样得到的迭代器可以当作尾后迭代器使用

创建ostream_iterator时，必须绑定一个流，同时可以提供第二个参数，一个字符串，输出每个元素都会打印这个字符串。

可以使用copy简化

10.4.3 反向迭代器

递增会移动到前一个；递减会移动到后一个

假定一个名为line的string，内是用逗号分隔的单词列表，现打印最后一个元素

10.5 泛型算法结构

算法要求的迭代器操作分为5个迭代器类别(iterator category)：

输入迭代器(input iterator)：可以读取序列中元素，支持

  • 用于比较两个迭代器的相等和不等运算
  • 用于推进迭代器的前置和后置递增运算(++)
  • 用于读取元素的解引用操作
  • 箭头运算符(->)，等价于(*it).member

输出迭代器(output iterator)：输入上的补集，只写不读

  • 用于推进迭代器的前置和后置递增运算(++)
  • 解引用运算符

前向迭代器(forword iterator)：可以读写，只能沿一个反向移动，支持所有输入输出迭代器的操作，且可以多次读写一个元素。

双向迭代器(bidirectional iterator)：可以正向/反向读写序列中元素。除了支持所有前向迭代器的操作，还支持前置和后置递减操作符(–)。

随机访问迭代器(random-access iterator)：常量时间内访问序列中任意元素。除了支持双向迭代器，还支持

  • 用于比较相对位置的关系运算符(<、<=、>、>=)
  • 迭代器和一个整数值的加减运算，结果是前进或后退整数个位置
  • 两个迭代器的减法，得到距离
  • 下标运算(iter[n])，与*(iter[n])等价

10.6 特定容器算法

头文件

list和forward_list成员函数版本算法：

  • lst.merge(lst2)：将来自lst2的元素并入lst。lst和lst2必须是有序的(使用<运算符)。合并之后，元素从lst2中删除

  • lst.merge(lst2, comp)：使用特定的比较操作(可以传入比较器)

  • lst.remove(val), lst.remove_if(pred)：调用erase删除给定值或者令一元谓词为真的每个元素

  • lst.reverse()：反转顺序

  • lst.sort()：使用 < 排序

  • lst.sort(comp)：用给定的操作排序

  • lst.unique()：调用erase删除同一个值的连续拷贝，即只存在一个给定值

  • lst.unique(pred)：使用给定的二元谓词

链表还有splice算法，为链表独有

lst.splice(args)或者flst.splice_after(args)

参数 说明
(p, lst2) p是指向lst中元素，或一个指向flst首前位置。
将lst2的所有元素移动到lst中p之前的位置或者flst中p之后的位置。
lst2中元素将删除。
lst和lst2必须是同类型链表，且不能是同一个
(p, lst2, p2) p2指向lst2中元素。
将p2指向的元素移动到lst中，或者p2之后的元素移动到flst中
lst2可以和lst是一个链表
(p, lst2, b, e) b和e必须表示lst2的合法范围
将给定范围的元素从lst2移动到lst或flst
lst2可以和lst是一个链表，但不能指向范围中元素

链表特有的操作会改变容器。
