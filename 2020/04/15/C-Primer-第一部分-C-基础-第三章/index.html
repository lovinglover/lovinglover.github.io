C++ Primer 第一部分 C++基础

第三章 字符串、向量和数组

3.2 标准库类型string

标准库类型string表示可变长的字符序列

3.2.1 定义和初始化string对象

直接初始化和拷贝初始化

使用等号(=)初始化一个变量实际上执行的是拷贝初始化(copy initialization)编译器把等号右侧的值拷贝到新创建的对象中，反之不用等号如string s3("lovinglover")则是直接初始化(direct initialization)。

3.2.2 string对象的操作

读写string对象

两个输入，以空白作为分隔，输入hello world ，”hello”给s1，”world”给s2；输出为”helloworld”

getline读取整行

getline()的参数是一个输入流和一个string对象，从给定的输入流中读入内容直到换行符(换行符也被读入)，然后存入string对象中(这里换行符不存)。

getline一遇到换行符就停止读取操作并返回结果，若一开始就是换行符则是空string

empty和size

empty()根据string对象是否为空返回一个布尔值，调用方法s1.empty()

size()返回string对象的长度(即字符的个数，不包括最后的空字符)，调用方法s1.size()

string::size_type类型

size()函数返回的是一个string::size_type类型的值，是一个无符号整型，最好不要和带符号数混用。

比较string对象

比较时逐一比较string对象中的字符，并对大小写敏感

  1. 若两个string对象长度不同，且较短的每个字符都与较长的在对应位置上相同，则较短的小
  2. 若在某些位置上不同，比较结果是第一对相异的字符比较结果
  3. 相等的条件是长度相同且对应位置上每个字符都相同

比较结果：s1小于s2；s2小于s3

两个string对象相加

string s3 = s1 + s2意思是把s2拼接到s1后面生成一个新的string对象并赋值给s3，这里s1和s2都不会发生改变，s1 += s2等价于s1 = s1 + s2

字面值和string对象相加

加法运算符两边至少有一个是string类型的对象

函数名 作用
isalnum(c) 当c是字母或者数字时为真
isalpha(c) 当c是字母时为真
iscntrl(c) 当c是控制字符时为真
isdigit(c) 当c是数字时为真
isgraph(c) 当c不是空格但可以打印时为真
islower(c) 当c是小写字母时为真
isprint(c) 当c是可以打印的字符时为真(空格或具有可视形式)
ispunct(c) 当c时标点符号时为真

isupper(c) 当c是大写字母时为真
isxdigit(c) 当c时十六进制数字时为真
tolower(c) 如果c是大写字母，输出对应的小写，否则原样输出
toupper(c) 如果c是小写字母，输出对应的大写，否则原样输出

使用基于范围的for语句处理每个字符

C++11提供的新语句，范围for(range for)，遍历给定序列中的每个元素并对其执行某种操作

expression是一个对象，表示某个序列，declaration定义一个变量用于访问序列中的基础元素，每次迭代，会被初始化为expression的下一个元素值

使用下标处理一部分字符

3.3 标准库类型vector

标准库类型vector表示对象的集合，其中所以对象类型相同。每个对象有一个索引用于访问，也叫容器(container)

vector是一个类模板，模板本身不是类或者函数。编译器根据模板创建类或者函数的过程称为实例化，使用模板时，需要指出将其实例化为何种类型

vector对象可以动态增长

3.3.1 定义和初始化

常用的定义方法

注意圆括号()和花括号{}的区别

圆括号提供的值用来构建vector对象的，一个值是元素个数，两个值表示元素个数和初值

花括号提供的值尽量用来初始化，作为初值，除非提供的值无法用来初始化

这里10不是string类型，无法作为string类型vector的初值

3.3.2 向vector对象中添加元素

直接初始化vector对象的方式适用于三种情况：

  1. 初始值已知且数量较少
  2. 初始值是另一个vector对象的副本
  3. 所有元素的初始值都一样

更常见的是先创建一个空的vector对象，再需要时使用push_back()函数向其中添加元素(添加到vector的尾端)

3.3.3 其他vector操作

函数名 作用
v.empty() 若v为空，返回真；否则返回假
v.size() 返回v中元素的个数
v.push_back(t) 向v的尾端添加元素t
v[n] 返回v中第n+1个位置上的元素引用
v1 = v2 v2中元素的拷贝替换v1中的元素
v1 = {a, b, c…} 列表中元素的拷贝替换v1中的元素
v1 == v2, v1 != v2 当且仅当元素数量且对应位置元素值都相同才相等
<, <=, >, >= 按字典顺序比较

3.4 迭代器(Iterator)介绍

可以通过下标运算符访问string对象和vector对象，迭代器是更加通用的机制，标准库中定义了很多容器，不是所有容器都支持下标运算符。

迭代器类似指针，提供对对象的简介访问，有有效和无效之分。

3.4.1 使用迭代器

与指针不同，获取迭代器不是使用取地址符，每个有迭代器的类型同时拥有返回迭代器类型的成员：begin、end。

begin返回指向第一个元素(或者字符)的迭代器；end返回指向容器(或string对象)”尾元素下一个位置(one past the end)”的迭代器，该迭代器指向的是本不存在的元素，表示处理完迭代器中所有元素。

若容器为空，begin和end返回的都是尾后迭代器

迭代器运算符

任意标准容器迭代器支持的运算符

运算符 作用
*iter 返回迭代器iter所指元素的引用
iter->mem 解引用iter并获取该元素的名为mem的成员，等价于(*iter).mem
++iter 令iter指向容器中下一个元素
–iter 令iter指向容器中上一个元素
iter1 == iter2, iter != iter2 当两个迭代器指向的是同一个元素或是同一个容器的尾后迭代器，相等；反之不等

执行解引用的迭代器必须合法并确实指向某个元素，所以end返回的迭代器无法执行递增和解引用操作

这里for循环中使用了!=，因为所有标准库容器的迭代器都定义了==和!=，只有string和vector等少数定义了<，所以使用迭代器和!=是更加通用的方法。

cbegin和cend

C++11新标准引入的新函数，作用同begin和end，但无论容器对象本身是不是常量，返回的都是const_iterator(只能读取不能修改)

解引用和成员访问操作

解引用迭代器获得其指向的对象，若该对象是类，则需要进一步访问其成员，如vector对象是字符串，需要检查其是否为空

C++引入了箭头运算符(->)简化上面问题，箭头运算符把解引用和成员访问结合在一起

某些对vector对象的操作会使迭代器失效

vector的动态增长有些副作用：

  1. 不能再范围for循环中向vector对象添加元素
  2. 任何可能改变vector对象容量的操作如push_back会导致该vector对象的迭代器失效

谨记：但凡是使用了迭代器的循环体，都不要向迭代器所属的容器添加元素

3.4.2 迭代器运算

vector和string除了支持标准容器迭代器的运算符外，还提供了一些额外的运算符

  1. iter + n、iter - n、iter += n、iter -= n；迭代器移动若干距离
  2. iter1 - iter2；得到两个迭代器之间的距离
  3. <、>、<=、>=；iter1指向的位置在iter2指向的位置之前，则iter1 < iter2

使用迭代器实现二分搜索

3.5 数组

数组类似于vector，也是存放相同类型的对象的容器，但是数组大小确定不变

3.5.1 定义与初始化

数组不允许拷贝和赋值

部分编译器支持数组的赋值，这就是所谓的编译器扩展(compiler extension)，但尽量避免使用非标准特性

复杂的数组声明

定义存放指针的数组比较简单，定义数组的指针或者引用就复杂些

3.5.3 指针和数组

用到数组名的地方，编译器会自动替换为指向数组首元素的指针：

指针也是迭代器

string和vector的迭代器支持的运算，数组的指针全部支持

标准库函数begin和end

这两个函数定义在iterator头文件中

和容器中的成员函数功能类似，begin返回指向数组首元素的指针，end返回指向数组尾元素下一个位置的指针

下标和指针

只要指针指向的是数组中的元素(或者数组中尾元素的下一个位置)，都可以执行下标运算

使用数组初始化vector对象

只需要指明拷贝区域的首元素地址和尾后地址

需要注意，第二个参数指向的是需要拷贝的尾元素的下一个位置

书本建议：尽量使用标准库类型而非数组，即尽量使用vector和迭代器，避免使用数组和指针
