C++ Primer 第二部分 C++标准库

第九章 顺序容器

一个容器就是一些特定类型对象的集合。顺序容器(sequential container)为程序员提供了控制元素存储和访问顺序的能力。这种顺序不依赖于元素的值，而是与元素加入容器时的位置对应。

9.1 顺序容器概述

下表列出了标准库中的顺序容器，使用顺序容器都提供了快速顺序访问的元素的能力。但是，这些容器在一些方面有不同的性能折中

  • 向容器添加或从容器中删除元素的代价
  • 非顺序访问容器中元素的代价
容器类型 说明
vector 可变大小数组，指出快速随机访问。在尾部之外的位置插入或者删除元素可能很慢
deque 双端队列。支持快速随机访问，在头尾位置插入/删除元素速度很快
list 双向链表。只支持双向顺序访问，在list中任何位置插入/删除操作速度都很快
forward_list 单项链表。只支持单向顺序访问，在链表任何位置插入/删除操作速度都很快
array 固定大小数组。支持快速随机访问，不能添加或者删除元素
string 与vector类似的容器，专门用来保存字符。随机访问快，在尾部插入/删除速度快

确定使用哪种顺序容器

  • 除非你有很好的利用选择其他容器，否则应该使用vector
  • 如果你的程序有很多小的元素，且空间的额外开销很重要，则不要使用list或者forward_list
  • 如果程序要求随机访问元素，应该使用vector或deque
  • 如果程序要求在容器的中间插入或删除元素，应该使用list或forward_list
  • 如果程序需要在头尾位置插入删除元素，但不会在中间位置插入删除元素，使用deque
  • 如果程序只有在读取输入时才需要在容器中间位置插入元素，随后需要随机访问元素，则
    • 首先，确定是否真的要在容器中间位置添加元素。当处理输入数据时，通常可以很容易向vector追加数据，然后调用标准库的sort函数来重排容器中元素，从而避免在中间位置添加元素
    • 如果必须在中间位置插入，考虑在输入阶段使用list，一旦输入完成，将list中内容拷贝到一个vector中

一般来说，应用中站主导地位的操作(访问操作还是插入/删除)决定了容器类型的选择。

如果不确定使用哪种容器，可以在程序中只使用vector和list的公共操作：迭代器，不用下标操作，避免随机访问。这样在必要时选择使用vector和list都方便。

  • 还有一些操作只适用于一小部分容器

一般来说，每个容器都定义在一个头文件中，文件名和类型名相同

对容器可以保存的元素类型的限制

顺序容器几乎可以保存任意类型的元素。特别是，容器的元素是另外一个容器

9.2.1 迭代器

迭代器范围

迭代器范围的概念是标准库的基础。

一个迭代器范围(iterator range)由一对迭代器表示，两个迭代器分别指向同一个容器中的元素或者尾元素之后的位置。这种范围是左闭合区间：[begin, end)

使用左闭合范围处理for循环

9.2.2 容器类型成员

大多数容器还提供反向迭代器，与正向迭代器相比，各种操作的含义也颠倒。

类型别名：可以在不了解容器中元素类型的情况下使用它。如若需要元素类型，可以使用value_type。使用这些类型必须显示使用

9.2.3 begin和end成员

begin和end有其他版本：带r的返回反向迭代器；以c开头的返回const迭代器

9.2.4 容器定义和初始化

每个容器类型都定义了一个默认构造函数。除array之外，都会创建一个指定类型的空容器，且都可以接受指定容器大小和元素初始值的参数。

将一个容器初始化为另外一个容器的拷贝

两种方法：直接拷贝整个容器；拷贝由迭代器对指定的元素范围(array除外)

两个容器的类型及其元素类型必须匹配；但当通过迭代器拷贝时，不要求容器类型相同，且元素类型也可以不同，能转换就可以

标准库array具有固定大小

当定义一个array时，除了指定元素类型，还要指定容器大小。

默认构造的array是非空的，它包含了与其大小一样多的元素，这些元素都被默认初始化，若这些元素是类，则该类必须由默认构造函数。

array可以拷贝或赋值

9.2.5 赋值和swap

赋值运算将左边容器中的全部元素替换为右边容器中元素的拷贝，若大小不同，替换后与右边容器原大小一致。

赋值 操作 说明
c1 = c2 c1中元素全部替换为c2中元素的拷贝
c = {a, b, c…} c1中元素替换为初始化列表中元素的拷贝(array除外)
swap(c1, c2)、c1.swap(c2) 交换c1、c2，c1和c2必须类型相同，swap操作比拷贝快一些
seq.assign(b, e) 将seq中元素替换为迭代器b、e范围的元素，迭代器b、e不能指向seq
seq.assign(il) seq中元素替换为初始化列表中il的元素
seq.assign(n, t) 将seq中元素替换为n个t

assign操作不适用于关联容器和array

使用assign(仅顺序容器)

顺序容器(除array)定义了成员assign，将用参数指定的元素(的拷贝)替换左边容器中的所有元素

如可以将vector中的一段char*赋予一个list中的string

assign的第二个版本接受一个整型值和一个元素值

使用swap

swap骄傲换两个相同类型容器的内容。

执行后，svec1包含24个string元素，svec2包含10个string元素。除array之外，交换两个容器操作很快，因为只交换了内部的数据结构

除array外，swap不对任何元素进行拷贝、删除或插入操作，因此可以保证在常数时间内完成

元素不会移动意味着，除string外，指向容器的迭代器、引用和指针在swap之后都不会失效。仍然指向之前的元素，但是swap后，这些元素不属于之前的容器。iter1之前指向svec1[3]，swap后指向svec2[3]。但是string类型会导致迭代器、引用和指针失效。

swap两个array会真正交换元素，时间与数目成正比。因此，对于array，swap后，指针、引用和迭代器所绑定的元素不变，但是值已经和另一个array中对应的元素值交换。

推荐使用非成员版本的swap()

9.2.6 容器大小操作

每个容器类型(除forward_list)有三个与大小相关的操作

  • size()。返回容器中元素的数目
  • empty()。当size()为0时返回true，否则返回false
  • max_size()。返回一个大于或等于该类型容器所能容纳的最大元素数的值

forward_list支持max_size()和empty()

9.2.7 关系运算符

左右两边必须是相同类型的容器，且保存相同类型的元素。

比较规则(类似string的比较)：

  • 如果两个容器具有相同大小且所有元素两两对应相等，则这两个容器相等；否则不等
  • 如果两个容器大小不等，但较小容器中每个元素都等于较大容器中的对应元素，则较小容器小于较大容器
  • 如果两个容器都不是另外一个容器的前缀子序列，则比较结果取决于第一个不相等的元素比较结果

9.3.6 容器操作可能使迭代器失效

向容器添加元素后：

  • 如果容器是vector或string, 且存储空间被重新分配，则指向容器的迭代器、指针和引用都会失效。如果存储空间未重新分配，指向插入位置之前的元素的迭代器、指针和引用仍有效，但指向插入位置之后元素的迭代器、指针和引用将会失效。
  • 对于deque，插入到除首尾位置之外的任何位置都会导致迭代器、指针和引用失效。如果在首尾位置添加元素，迭代器会失效，但指向存在的元素的引用和指针不会失效。
  • 对于list和forward_list，指向容器的迭代器(包括尾后迭代器和首前迭代器)、指针和引用仍有效。

删除元素后：

  • 对于list和forward_ list，指向容器其他位置的迭代器(包括尾后迭代器和首前迭代器)、引用和指针仍有效。
  • 对于deque，如果在首尾之外的任何位置删除元素，那么指向被删除元素外其他元素的迭代器、引用或指针也会失效。如果是删除deque的尾元素，则尾后迭代器也会失效，但其他迭代器、引用和指针不受影响:如果是删除首元素，这些也不会受影响。
  • 对于vector和string,指向被删元素之前元素的迭代器、引用和指针仍有效。注意:当我们删除元素时，尾后迭代器总是会失效。

建议：每次改变容器的操作之后都要正确的重新定位迭代器

9.4 vector对象如何增长

vector将元素连续存储。容器扩大时，必须分配新的内存空间来存储已有元素和新元素，移动旧元素，添加新元素，释放旧空间。

当不得不获取新的空间时，vector和string的实现会分配比新的空间需求更大的内存空间，避免每次添加新元素都要重新分配。这种分配策略的扩张操作比list和deque还快。

操作 说明
c.shrink_to_fit() 将capacity()减少到与size()相同
c.capacity() 不重新分配空间情况下，c可以保存的元素
c.reserve(n) 分配至少能容纳n个元素的内存空间
  • shrink_to_fit()只适用于vector、list和deque
  • capacity()和reserve()只适用于vector和string

调用reserve后，capacity会大于等于传递给reserve的参数

可以使用shrink_to_fit要求回退空间，但不一定一定会回退

注意capacity和size的区别

a.empty() 空返回true，否则返回false
a.size() 返回a中元素数目
swap(a, b)
a.swap(b)		 交换a和b的内容，a和b必须有相同类型，底层容器类型也要相同

9.6.1 定义一个适配器

每个适配器都有两个构造函数：默认构造函数创建一个空对象；接受一个容器的构造函数拷贝该容器进行初始化

默认情况下，stack和queue是基于deque实现的；priority_queue是在vector上实现的。可以重载默认容器类型

  • 适配器要求容器具有添加删除和访问尾元素的能力，不能使用array和forward_list构造适配器
  • stack只要求push_back、pop_back和back重载，可以使用除array和forward_list之外任何容器构造
  • queue要求back、push_back、front和push_front，可以用list和deque，不能用vector
  • priority_queue除了front、push_back和pop_back外，还要随机访问能力，可以用vector或deque，不能用list

9.6.2 栈适配器

stack适配器定义在stack头文件里。stack默认基于deque，也可以用list或vector

先进后出(first-in，last-out，FILO)

操作 说明
s.pop() 删除栈顶元素，但不返回
s.push(item)
s.emplace(args)		 创建一个新元素压入栈顶
s.top() 返回栈顶元素，但不弹出

只能使用适配器的操作，不能使用底层容器的操作

9.6.3 队列适配器

queue和priority_queue定义在queue头文件里。queue默认基于deque实现，也可以用list或vector；priority_queue默认基于vector，也可以用deque

操作 说明
q.pop() 删除queue的首元素或priority_queue优先级最高的元素，但不返回
q.front()
q.back()		 返回首元素或尾元素，不删除。只适用于queue
q.top() 返回优先级最高的元素，不删除。只适用于priority_queue
q.push(item)
q.emplace(args)		 在queue末尾或者priority_queue中恰当位置创建一个元素

queue先进先出(first-in，first-out，FIFO)

priority_queue为元素建立优先级，默认在元素类型上使用<运算符确定优先级，可以重载。
