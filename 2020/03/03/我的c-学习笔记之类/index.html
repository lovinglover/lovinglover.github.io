我的c++学习笔记之类

发表于 2020-03-03 分类于 c++学习笔记

类基础

面向过程与面向对象

面向过程的程序中函数是构成程序的基本单位，每实现一个新功能就新写一个函数，之后调用就可以。

面向对象编程就不是为了实现功能而写个函数。以狗为例，狗有很多属性，如品种、体型大小、毛发、颜色，也会有很多动作，跳、吠，描述一个对象时应该包括这些要素。

任何一个对象都应该有属性和行为这两个要素，一个对象也是由多个属性和多个行为构成的。

在面向对象的编程中，对象由数据和函数两个部分构成，数据就是属性，函数就是行为。

类的声明

类是对象的类型，用来表示一类具有共同属性和行为的类型，对象是类的实例，真真实实的变量，占用存储空间，而类不占空间。

定义一个基本的学生类，包含姓名、学号、性别、年龄基本属性

成员访问限定符

public 公有的，公开的，既可以被本类中的成员函数引用，也可以被类的作用域的其他函数所引用
private 私有的，只能被本类的成员函数调用，类外部无法调用(友元类除外)
protected 受保护的，可以被派生类的成员函数访问

上面的学生类，可以定义为只有姓名可以公开访问

或者

默认的就是private，成员访问限定符的作用范围是从它的下一行开始到结束或者出现另外一个成员访问限定符。

需要注意的是，用类定义的多个对象之间彼此独立，每个对象都拥有自己的一套成员变量。

成员函数

普通成员函数

类的普通成员函数跟普通的函数没什么区别，都有参数、返回值，区别在于必须通过类的实例化对象去调用。但是普通成员函数不属于对象，不同于成员变量，成员函数在类中只有一个备份。

存取器函数

针对私有成员变量，无法在类外被调用访问，必须提供一个public的函数来访问、设置，不然这个私有变量就没有意义了，

这些用来设置或获取私有成员变量的函数就称为存取器。

静态成员

静态成员变量

上面说过，每个对象都有自己的一套独立的成员变量，但若是有一个变量所有的对象都是一样的，则没有必要让每个对象都保存一份，这样浪费空间，而且不便于修改。以上面的Student类为例，有个headmaster变量，假设这些学生都来自一个学校，那么这个变量一定是一样的，这个时候可以声明为静态成员变量。

关于静态成员变量

  1. 所有的对象都可以访问，且值是一样的
  2. 在内存中只占用一份存储空间
  3. 若一个对象更改它，其它成员访问时也是被更改后的
  4. 只要类定义了静态成员变量，就占存储空间，不管有没有实例化对象，静态成员变量属于类
  5. 若声明在类外则不需要static关键字
  6. 既可以通过对象名调用，也可以通过类名调用
  7. 若静态成员变量设置为private私有的，则只能通过public的静态成员函数调用

静态成员函数

  1. 不属于对象，属于类
  2. 可以通过类名调用，也可以通过对象调用
  3. 可以直接访问静态成员变量，二者作用域相同，属于类
  4. 没有this指针

成员函数的声明与实现

声明一定要在类中，但是实现可以放在类外。

函数的实现可以全部放在类中，但若类很复杂，会导致类的代码很长，不便于阅读；比较好的写法是将将类的声明写在类中，实现写在类外。

写在类外必须要用作用域修饰符::，Student::

也可以把声明写在头文件中，实现写在cpp文件中，这样只要在需要使用这个类的文件中包含该头文件。

inline内联函数

使用inline关键字声明的函数叫做内联函数，可以使函数在被调用时将这段代码直接嵌入到调用的地方，可以减少函数调用的开销。

函数调用需要一些准备工作：将实参、局部变量、返回地址以及若干寄存器压入栈中(保护现场)，之后才能执行函数体中的代码；执行后要清理现场，将之前压栈的数据出栈(恢复现场)，继续执行调用位置之后的代码。

如果函数代码多，执行时间长，这些调用机制占用的时间便可以忽略；若函数只有一两条语句，那么大多数的时间都浪费在调用机制上，这是就可以使用inline内联函数。

使用

默认情况下，类体中直接实现的函数会被视为inline内联函数，上例Student类中的int get_age()、char get_sex()、int get_num()

类外直接使用inline声明就可以

注意

一般来说，只将短小的、频繁调用的函数声明为内联函数

构造函数和析构函数

构造函数

一种特殊的成员函数，名字与类名相同，没有返回值，不需要显示调用，用户也不应该调用，在创建对象时自动执行，但要注意，不是它来创建对象

如果用户没有自定义自己的构造函数，会自动生产一个默认的空构造函数

用这种方法会使构造函数看起来更简洁些

析构函数

析构函数也是一种特殊的成员函数，作用与构造函数相反，只有在对象的生命周期结束时会被自动调用，同样要注意不是它销毁对象，名字与类名相同，前面带上一个~

默认情况下也会自动生成一个空的析构函数

会自动调用析构函数的几种情况

  1. 在一个函数中定义了一个局部变量的对象，当这个函数结束执行就会调用
  2. 全局变量或者static类型的一般在程序退出时
  3. 使用new创建的对象，只有delete或者程序退出
